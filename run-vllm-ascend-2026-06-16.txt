===== user =====
--shm-size是什么参数？

===== assistant =====
`--shm-size` 是 Docker 给容器分配 `/dev/shm` 共享内存大小的参数。

它不是 NPU 显存，也不是模型权重占用的内存；它用的是宿主机 RAM，给容器里的多进程通信、缓存、worker 之间共享数据用。Docker 默认 `/dev/shm` 只有 `64MB`，跑 vLLM、PyTorch、多进程推理时太小，容易出现奇怪问题，比如 worker 崩、`Bus error`、进程通信失败、初始化卡住。

可以这样理解：

```bash
--shm-size=16g
```

意思是：容器内 `/dev/shm` 最大可以用 16GB。

进容器后可以看：

```bash
df -h /dev/shm
```

你之前命令里用过：

```bash
--shm-size=1g
```

这个对简单测试可能够，但你要跑 vLLM，尤其多卡/大模型，我建议先用：

```bash
--shm-size=16g
```

如果宿主机内存很大，也可以 `32g`。如果宿主机内存紧张，就先 `4g` 或 `8g`。

一句话：`--shm-size` 是容器共享内存大小；vLLM 这类多进程推理服务建议比默认值大很多，先用 `16g` 比较稳。

===== assistant =====
最推荐的路线是：Windows 的 `mfe` 文件夹先传到公司服务器宿主机的 `~/lhy/vllm-ascend0616/mfe`，然后容器通过你已有的目录挂载直接看到它。不要优先用 `docker cp`，因为容器删了文件也容易丢；挂载目录更适合开发。

第 1 步：在公司服务器宿主机上确认挂载目录存在。

```bash
mkdir -p ~/lhy/vllm-ascend0616
```

第 2 步：在 Windows PowerShell 里，把本机项目传到服务器。

如果你的 Windows 项目在 `D:\lihanyang\mfe`，用：

```powershell
scp -r D:\lihanyang\mfe 用户名@服务器IP:~/lhy/vllm-ascend0616/
```

传完以后，服务器上应该有：

```bash
~/lhy/vllm-ascend0616/mfe
```

如果目录很大，第一次用 `scp -r` 可以，之后增量同步更建议用 `rsync`。Windows 如果有 Git Bash/WSL，可以用：

```bash
rsync -av --delete /d/lihanyang/mfe/ 用户名@服务器IP:~/lhy/vllm-ascend0616/mfe/
```

第 3 步：启动 docker 时挂载 `~/lhy/vllm-ascend0616`。

你之前的 docker 命令里保留这一行：

```bash
-v ~/lhy/vllm-ascend0616:/root/lhy/vllm-ascend0616 \
```

这样宿主机：

```bash
~/lhy/vllm-ascend0616/mfe
```

在容器里就是：

```bash
/root/lhy/vllm-ascend0616/mfe
```

第 4 步：进容器后检查文件。

```bash
cd /root/lhy/vllm-ascend0616
ls
cd mfe
ls
```

能看到你的 `mfe` 项目文件，就说明传输和挂载成功了。

如果容器已经在运行，也可以临时拷进去，但这不是首选：

```bash
docker cp ~/lhy/vllm-ascend0616/mfe vllm-ascend:/root/lhy/vllm-ascend0616/mfe
```

不过更好的方式还是：先把文件放在宿主机挂载目录里，再启动容器。这样你以后更新 Windows 的 `mfe`，重新 `scp/rsync` 到宿主机，容器里马上就能看到。

你现在可以先做两件事：

1. Windows PowerShell 跑 `scp -r D:\lihanyang\mfe 用户名@服务器IP:~/lhy/vllm-ascend0616/`
2. 容器里跑 `ls /root/lhy/vllm-ascend0616/mfe`

我不要一串命令，你一步步告诉我怎么做

===== assistant =====
好，我们按“每一步做完再看下一步”的方式来。你现在先别急着跑 `vllm serve`。

第 1 步：在宿主机确认有几张卡。

```bash
npu-smi info
ls /dev/davinci[0-9]*
```

你要看的是卡号，比如 `/dev/davinci0` 到 `/dev/davinci7` 就是 8 张，之后 `tensor-parallel-size` 最多可以设 8。

第 2 步：先只启动容器，不跑模型。

如果你是 8 张卡，用这个版本。它会把 0-7 全部挂进去：

```bash
docker run --rm -it \
  --name vllm-ascend \
  --pull=never \
  --net=host \
  --shm-size=16g \
  --device /dev/davinci0 \
  --device /dev/davinci1 \
  --device /dev/davinci2 \
  --device /dev/davinci3 \
  --device /dev/davinci4 \
  --device /dev/davinci5 \
  --device /dev/davinci6 \
  --device /dev/davinci7 \
  --device /dev/davinci_manager \
  --device /dev/devmm_svm \
  --device /dev/hisi_hdc \
  -v /usr/local/dcmi:/usr/local/dcmi:ro \
  -v /usr/local/bin/npu-smi:/usr/local/bin/npu-smi:ro \
  -v /usr/local/Ascend/driver/lib64/:/usr/local/Ascend/driver/lib64/:ro \
  -v /usr/local/Ascend/driver/version.info:/usr/local/Ascend/driver/version.info:ro \
  -v /etc/ascend_install.info:/etc/ascend_install.info:ro \
  -v /root/.cache:/root/.cache \
  -v ~/lhy/vllm-ascend0616:/root/lhy/vllm-ascend0616 \
  -w /root/lhy/vllm-ascend0616 \
  -p 8000:8000 \
  quay.io/ascend/vllm-ascend:v0.20.2rc1 bash
```

如果你不是 8 张卡，就先告诉我 `ls /dev/davinci[0-9]*` 的输出，我帮你改这一段。

第 3 步：进容器后确认挂载成功。

在容器里执行：

```bash
npu-smi info
ls /dev/davinci[0-9]*
python -c "import vllm; print(vllm.__version__)"
```

这三条都正常，才进入下一步。

第 4 步：先用 1 张卡跑最小模型。

先不要所有卡一起上。先验证链路：

```bash
export ASCEND_RT_VISIBLE_DEVICES=0
export VLLM_USE_MODELSCOPE=True

vllm serve Qwen/Qwen3-0.6B \
  --host 0.0.0.0 \
  --port 8000 \
  --served-model-name qwen3-0.6b \
  --tensor-parallel-size 1
```

第 5 步：宿主机另开一个终端测试服务。

```bash
curl http://127.0.0.1:8000/v1/models
```

能返回模型列表，就说明 vLLM 服务跑起来了。

第 6 步：再考虑多卡。

如果你要用 8 张卡，把容器里的启动改成：

```bash
export ASCEND_RT_VISIBLE_DEVICES=0,1,2,3,4,5,6,7
```

然后 `vllm serve` 里改：

```bash
--tensor-parallel-size 8
```

关键点：Docker 挂载全部卡，只是让容器“看得到”全部卡；vLLM 真正用几张卡，是由 `ASCEND_RT_VISIBLE_DEVICES` 和 `--tensor-parallel-size` 决定的。

你现在先做第 1 步和第 2 步。第 3 步里如果 `npu-smi info` 在容器内也正常，我们再继续跑模型。
